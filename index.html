Horario permitido de carga

Compartir Locales/Config en GitHub

Código, nombre, provincia, responsable, calendario y fuentes se guardan en el repo público para que cualquiera con el link vea todo actualizado sin configurar de cero. Los emails nunca se suben — quedan solo en este navegador.

¿Cómo crear el token? GitHub → foto de perfil → Settings → Developer settings → Personal access tokens → Fine-grained tokens → New token. Repository access: solo este repo. Permissions → Contents: Read and write. Copiá el token y pegalo arriba (empieza con "github_pat_" o "ghp_").

Envío automático de reclamos por mail

Todos los días a las 23:35 se revisa quién no completó su control y se le manda un reclamo por mail (ver pestaña "Respuestas" para el detalle de cada caso). Los emails de cada local se guardan encriptados como un GitHub Secret — ni siquiera vos podés volver a leerlos desde GitHub, solo actualizarlos.

Usa el mismo token de arriba. Además de Contents: Read and write, necesita el permiso Secrets: Read and write (lo agregás editando el token en GitHub, o creando uno nuevo).
